# Documentação do Projeto

## Fontes normativas

1. ADR `ACCEPTED` mais recente;
2. Contexto Mestre v2.3;
3. especificação da sprint;
4. documentação funcional;
5. guia visual para decisões de interface.

## Estado real

O código, as migrations e os testes executáveis definem o que está realmente implementado.

Classificações:

- IMPLEMENTED;
- PLANNED;
- DIVERGENT;
- DEPRECATED;
- BLOCKED.

## Diretórios

- `adr/`: decisões arquiteturais;
- `sprints/`: execução e evidências;
- `ui/`: design system e regras visuais;
- `generated/`: índices automáticos;
- `runbooks/`: operação local e produção;
- `domains/`: preenchido a partir da Sprint 1.
